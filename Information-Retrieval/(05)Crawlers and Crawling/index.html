(05) Crawlers and Crawling

Lectrue in 01/22/2018

[TOC]

1. Web Crawler

A Web Crawler (spider), is an Internet bot that systematically browses the World Wide Web, typically for the purpose of Web indexing (web spidering).

There are something related to web crawling that we need to care about:

  1. Crawling Quality: find the “best” pages
  2. Crawling Efficiency: avoid dups and near-dups
  3. Crawling Etiquette (Politeness): not visiting websites too many times in a short time (this may afftect websites’ performance)
  4. Crawling Coverage: what percent of web to cover?
  5. Crawling Freshness: pages may change…

2. Basic Crawling Process

    1. Start with a “seed” URL
    1. Fetch and parse a page
      • a) Download the page and store it into a database
      • b) Parse that page, extract its outgoing URLs
      • c) Put the extracted URLs to a queue (So crawling strategy could be BFS, DFS, Best First, …)
    1. Fetch each URL in the queue and repeat step (2)

For step 2b, notice that

  1. Links are not just aside in <a> tag, they could also apear in <frame>, <meta>, <script> and so on.
  2. Some <a> do not contain a link, such as <a name="example">
  3. Some <a> may lead to infinite loop

For step 2c, we could use BFS, DFS or some Heuristic Method to re-ordering URLs on the queue:

  1. Move forward URLs with larger In-degree
  2. Move forward URLs with larger PageRank

3. Some chanllenges for crawling:

  1. Malicious pages (spam, crawler trap such as infinite loop or visiting a page over and over again due to SessionID)
  2. Latency and bandwidth
  3. Duplicate pages (This is mainly because of site mirrors)
  4. Be polite

3.1 About Spams

Three generations of spam are listed (They are all outdated): [1]

  1. Pages that contain many repeated words to rank higher by word frequency
  2. Cloaking. Two Different pages/content with same URL. One for the search engine/bot and the other for human.
  3. Doorway Page. Doorway page is a keyword stuffed page which after a few minutes redirects to another page. Clicking on a single URL will open multiple links either simultaneously or sometimes later. Mostly you might experience these techniques live when going through any movie downloads (in a pirated way). Best example for this is: torrentz.

3.2 About Crawler Trap - SessionID [2]

A spider trap is when a spider re-spiders the same page over and over again, you can compare it to a maze (labyrinth).

The most known spider trap is Session ID’s, a Session ID is often used to keep track
of the visitors, and some sites puts a unique ID in the URL: An example is www.webmasterworld.com/page.php?id=264684413484654
Each user gets a unique ID and it’s often requested from each page.
The problem here is when Googlebot comes to the page, it spiders the page and
then leaves, it comes back to another page and it finds a link to the same page but since it has been given a different session id now, the link shows up as another URL.

This is one of the reasons why Googlebot is very very carefully when it spiders pages which uses the querystring “ID=”.

See more: Googlebot: Deepbot and Freshbot FAQ and Information

5 Avoid duplicate URLs

To avoid duplicate URLs, we:

  1. Normalize URLs;
  2. Determine whetehr a URL has already been seen.

5.1 Normalizing URLs

Normalizing is important. For example, http://www.google.com, https://www.google.com, google.com all point to the same page, but they have different hashcode.

Some ways to normalize URLs:

  1. Converting the scheme and host to lower case. HTTP://www.Example.com/ → http://www.example.com/
  2. Capitalizing letters in escape sequences. http://www.example.com/a%c2%b1b → http://www.example.com/a%C2%B1b
  3. Decoding percent-encoded octets of unreserved characters. http://www.example.com/%7Eusername/ → http://www.example.com/~username/
  4. Removing the default port. http://www.example.com:80/bar.html → http://www.example.com/bar.html

See more: URL normalization

5.2 Determine whetehr a URL has already been seen

Two ways to determining whetehr a URL has already been seen:

(1) Hash(on host) and Trie(on path)
  1. Hash on host/domain name
  2. Use trie to store and check paths. Nodes will be like “/cs”, “/viterbi”. Put a symbol “$” at the end of each word to make them different from almost identical elements.

For n URLs and at most k branches each layer, time complexity of determining whetehr a URL has been seen is $O(log_k{n})$ (Tree height).

If we have a huge amount of URLs, we need to use distributed systems, like consistent hashing to map similar URL to same machine. See more from Distributed Systems Part-1: A peek into consistent hashing!

(2) Use $\Delta$ between the current and previous URLs in sorted URLs
  • Sort URLs lexicographically,
  • Store them as a delta-encoded text file.

In this way, entries are stored as the difference $\Delta$ between the current and previous URL. Advanges of this method are that we could use binary search and required storage is reduced. But notice that restoring actual URL takes more time since we need to apply all $\Delta$s to the initial URL.

7. Multi-Threaded Crawling & Distributed Model

Three strategies of Coordination of Distributed Crawling

  1. Independent
  2. Dynamic Assignment: A central coordinator dynamically divides the web and assigns partitions.
  3. Static Assignment: Web is partitioned and assigned without a central coordinator before the crawl starts

Links from one partition to another in Static Assignment can be handled:

  1. Firewall mode: Processes do not follow any inter-partition link, but this actually works not bad -> good coverage with low cost
  2. Cross-over mode
  3. Exchange mode

8. Freshness

Web pages could be created, updated and deleted, so we need to periodically check crawled pages to see if there is any change. To do so, check Last-Modified response in HTTP header.

Be aware that when a page changes either at a low frequency or high frequency, we should not visit it too often. Fot the latter case, if a page changes very frequently, even if visit with short time interval, we still cannot guarantee it’s fresh. In fact, a way to improve freshness is to penalize the elements that change too often.
