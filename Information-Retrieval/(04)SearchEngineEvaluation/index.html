(04) Search Engine Evaluation

Lectrue in 01/17/2018

1. Precision and Recall

Machine Learning on Coursera [Week6 -> error-metrics-for-skewed-classes]:

PC \ AC 1 0
1 True Positive False Positive
0 False Negataive True Negataive

PC is Predicted Class, AC is Actual Class.
So, True Positive means for a data record whose class is 1 (positive), we predict is as 1. If we predict such data as 0, then this is False Positive.

In Information Retrieval, this form can be defined as:

Relevant Non-relevant
Retrieved True Positive (TP) False Positive (FP)
Not Retrieved False Negataive (FN) True Negataive (TN)

For example, if a retrieved page is relevant, then this is True Positive; if a retrieved page is not relevant, then this is False Positive. Retrieving this page means we predict it as positive, but this prediction may be true or may be false.

After defining these, we could define precesion and recall:

For web applications, Precision is more important than Recall

We could also use Set Notation to represent precesion and recall:

  1. A is set of relevant documents
  2. B is set of retrieved documents
Relevant Non-relevant
Retrieved $A \cap B$ $\bar{A} \cap B$
Not Retrieved $A \cap \bar{B}$ $\bar{A} \cap \bar{B}$

Recall is diﬃcult to measure on the web since we do not know really how many items are relevant.

By empirical confirmation (not theorem), precision decreases as either the number of docs retrieved (or recall) increases.

The Harmonic Mean tends strongly toward the least element of the list making it useful in analyzing search engine results

As a simple example, the harmonic mean of 1, 4, and 4 is 2, tends toward the least element 1.

3. Mean Average Precision

Mean average precision (MAP) for a set of queries is the mean of the average precision scores for each query. [wikipedia]

where Q is the number of queries.

For example, there are two queries, query1 and query2.

For query1, there is a relavant documents {a,b,c,d,e}. Calculate Recall / Precision at Fixed Positions, use 1 for revalent document and 0 for irrevanlent.

Average Precision for query1 is $\frac{1.0 + 0.67 + 0.5 + 0.44 + 0.5}{5} = 0.62$

For query2, there is a relavant documents {f,g,h}. Calculate Recall / Precision at Fixed Positions, use 1 for revalent document and 0 for irrevanlent.

So in this case, MAP = $\frac{0.62 + 0.44}{2} = 0.53$

4. Discounted Cumulative Gain

Discounted Cumulative Gain (DCG) measures the usefulness, or gain, of a document based on its position in the result list. The gain is accumulated from the top of the result list to the bottom, with the gain of each result discounted at lower ranks. [1]

The Premise of DCG is that highly relavant documents appearing lower in a search result list should be penalized as the graded relevance value is reduced logarithmically proportional to the position of the result. [2]
