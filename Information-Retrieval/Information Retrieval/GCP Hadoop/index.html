HW3 GCP Hadoop - InvertedIndex

GCP Hadoop - InvertedIndex

CSCI572第三次作业，Hadoop Exercise to Create an Inverted List，记录一下GCP Hadoop的使用。

写在最前面：

  1. 每次使用完毕后都去关闭billing账单以免超额。Navigation in the left side -> Billing -> Overview-> Select the project -> "Disable billing"
  2. 每次需要使用时启用billing后，需要到Dataproc->Clusters里手动开启所有的VM Instances(不会随着开启billing而自动开启)

1. 上传文件数据

需要生成倒排索引的文件是3036本书(txt)，在full_data文件下，需要将这个文件夹其上传到Cloud Storage staging bucket，bucket可以从下图中红色部分进入。

如果是少量文件，可以直接使用bucket中的UI->upload进行操作，但这里进行上传1G总量的3000本书时出现页面无法响应，所以使用SSH上传：

  1. 本地压缩full_data成full_data.zip
  2. GCP->Cluster->VM Instances中连接SSH，上传master node
  3. 解压，得到full_data文件夹
  4. 复制这个文件夹内容到bucket的full_data目录下：hadoop fs -cp ./full_data gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/full_data


2. 路径配置

  1. 第一次启动需要配置Java_Home环境变量:

exportPATH=${JAVA_HOME}/bin:${PATH}

  1. 之后每次打开新的SSH终端都要配置HADOOP_CLASSPATH路径:

export HADOOP_CLASSPATH=${JAVA_HOME}/lib/tools.jar


3. Java编译和生成Jar

每本书开头为书籍id，后面接正文内容，样例如下：

由此生成倒排索引的map-reduce代码较简单，与word count类似，略过。主要思路是将word cound的map中的<word,1>改为输出<word, docId>，reduce中使用一个hashMap记录word所出现的docId和次数：

GCP的SSH连接提供UI，可以vi编写代码InvertedIndex.java（mapper和reducer如果写成内部类，则注意用static，否则hadoop无法实例化，外部类则主要需要空构造方法），完成后编译和生成jar。

  1. 编译java

hadoop com.sun.tools.javac.Main InvertedIndex.java

  1. 生成jar（这里是写成内部类）

jar cf WordCount.jar InvertedIndex*.class

  1. 复制到bucket的jars目录下：

这里有两步，先复制这个jar文件，再复制到bucket里（新建jars文件夹用来存在），否则会提示 cp: './InvertedIndex.jar': No such file or directory

  1. hadoop fs -copyFromLocal InvertedIndex.jar
  2. hadoop fs -cp ./InvertedIndex.jar gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/jars

hadoop fs -copyFromLocal InvertedIndex.jar执行后就会复制这个jar，如果后面重新生成了jar并希望覆盖原有的，可以先删除，再复制：

hadoop fs -rm -r InvertedIndex.jar

hadoop fs -copyFromLocal InvertedIndex.jar

参考：How to overwrite the existing files using hadoop fs -copyToLocal command


4. 提交Job

现在bucket里已经有了需要的jar，可以提交job运行了。

Main class or jar填InvertedIndex

jar文件路径:gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/jars/InvertedIndex.jar

而后是两个java文件的参数args[0], args[1]，分别为输入、输出路径：

输入文件路径：gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/dev_data

输出文件路径：gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/output_data

注意输出路径需要事先不存在。


5. 合并最终结果

运行后结果会在bucket里，SSH进入master node合并Job的运行结果，输入：

hadoop fs -getmerge gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/output_data ./output.txt

最后复制到bucket，和之前复制jar一样

  1. hadoop fs -copyFromLocal ./output.txt
  2. hadoop fs -cp ./output.txt gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/

执行命令，按顺序：

  1. 导入环境，export HADOOP_CLASSPATH=${JAVA_HOME}/lib/tools.jar (否则会提示找不到…javac.Main)

  2. 新建文件，vi InvertedIndex.java

  3. 输入代码

  4. 编译文件，hadoop com.sun.tools.javac.Main InvertedIndex.java

  5. 导出jar， jar cf InvertedIndex.jar InvertedIndex*.class

  6. 复制jar，

    hadoop fs -rm -r InvertedIndex.jar (删除jar)

    hadoop fs -copyFromLocal InvertedIndex.jar

    hadoop fs -cp ./InvertedIndex.jar gs://dataproc-18f6f9bb-d9cc-45aa-a15f-65983bb22d82-us-west1/jars
