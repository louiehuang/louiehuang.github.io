访问速度优化(一)---图片及谷歌字体转存COS

     这个blog (http://hlyin.space) 一直是用chrome做测试和访问，刚搭建起来的时候打开速度比较正常，大概在2、3秒的样子，但这几天突然变得非常慢，需要等1、2分钟才能加载出来，想起来之前都是挂着ss，而现在没有开ss，觉得可能问题出在访问了被墙的资源，因而做些测试来查找并解决访问速度问题。
     比较奇怪的是，用firefox和safari访问速度都比较快，只有chrome打开极慢，打开chrome的开发者工具（快捷键F12），选择network项，然后在历史记录中清除缓存以避免影响测试结果，然后访问网站 (http://hlyin.space) ，如下图，加载时间为1.5分钟，在Time项中可以观察各项请求的耗时，找到了最耗时的项目，502网关错误，即红色项:

放大一点：



错误提示：

  1. Font from origin ‘http://fontstatic.useso.com‘ has been blocked from loading by Cross-Origin Resource Sharing policy: No ‘Access-Control-Allow-Origin’ header is present on the requested resource. Origin ‘http://hlyin.space‘ is therefore not allowed access. The response had HTTP status code 502.
  2. Get http://fontstatic.useso.com/s/montserrat/v6/zhcz-_WihjSQC0oHJ9TCYPk_vArhqVIZ0nv9q090hN8.woff2

     可以看出，问题出在zhcz-_WihjSQC0oHJ9TCYPk_vArhqVIZ0nv9q090hN8.woff2这个字体文件，502网关错误，这个字体的链接在http://fontstatic.useso.com中，页面相关代码如下：

     这里需要解释一下，blog使用了包括Open Sans等字体在内的谷歌字库字体，而谷歌由于被禁，字体链接不能访问而换成了360的CND，fonts.useso.com，像上面代码中http://fonts.useso.com/css?family=Open+Sans:400,600 (谷歌的为http://fonts.googleapis.com/css?family=Open+Sans:400,600)，然而检测结果表明即使换成360CND，这种web font的加载仍十分耗时。

     在这两个link里使用了访问受阻的字体，查看链接，zhcz-_WihjSQC0oHJ9TCYPk_vArhqVIZ0nv9q090hN8.woff2(无法获取)在第一个链接中被访问，cJZKeOuBrn4kERxqtaUH3aCWcynf_cDxXwCLxiixG1c.ttf (也无法获取)在第二个链接中被访问。



    

由此可以有两种解决方法：

  1. 如果不需要这个字库，则可以直接去掉这两个字体链接，即把这两条link语句去掉， 虽然是省事，但字体效果没有了的话还是不太可取；
  2. 将字体添加到自己的云存储中通过外链导入，或者镜像到本地，正确的处理方法，之一，当然应该是这样

     先直接删掉上面那两个link做一下测试，清除缓存后再用chrome测加载时间，结果如下图：

     加载时间为13.98秒，相比于1.5分钟有很大的改善，那么可以确定网页打开速度确实受选择的字体影响很大，为了继续使用选择的字体，将其下载并存放到云存储中。
     云对象存储之前用过七牛，但各个云存储供应商，如果不实名认证，能使用的资源非常有限，因为买服务器的时候在腾讯认证过，所以转用腾讯的云存储，使用比较方便，开通对象存储服务就可以，而且50G以内不收费，放图片、字体等资源的话足够了。
（1） 先下载所用字体，以http://fonts.useso.com/css?family=Open+Sans:400,600为例，打开后如下图，打开其中所有的url下载字体，http://fontstatic.useso.com/s/opensans/v13/cJZKeOuBrn4kERxqtaUH3T8E0i7KZn-EPnyo3HZu7kw.woff 和 http://fontstatic.useso.com/s/opensans/v13/MTP_ySUJH_bn48VBG8sNSnhCUOGz7vYGh680lGh-uXM.woff

（2） 在腾讯云对象存储的控制台新建一个bucket，我这里是gitimage，新建文件夹存放，上传刚刚下载的两个字体文件

（3） 本地新建一个css文件，我直接取名叫family=Open+Sans:400,600.css将http://fonts.useso.com/css?family=Open+Sans:400,600的内容复制进来，将url改成云存储中字体的链接，将这个css也存到云存储中，这时就可以用它的地址也替换link中原来的地址： <link href='//gitimage-10031767.file.myqcloud.com/blog_image/Theme_Image/family%3DOpen%2BSans400%2C600.css' rel='stylesheet' type='text/css'>



     这样就可以使用谷歌字库的字体了，但13.98秒的打开速度对于一个blog来说打开时间仍很长，继续…
     现在测试结果里最耗时的两项分别是我的图标和背景的加载，198KB的主页背景(bg.jpg)耗时10.13s，264KB的图标(firedoge.ico)耗时11.86s，讲道理的话这种大小是不会用这么多时间的，然而这两个图都是挂在github，国内访问确实会慢不少，因而也将其转存到国内的云存储中，把图片存进去后如下图：



     再次清除缓存后用chrome测试，结果如下，加载时间降至1.81s，虽然现实.ico图标耗时6s多，但整个blog加载出来不到2s，图标并不影响其他部分的显示。

     在firefox的开发者工具中测试，加载时间为1.01秒，也有改善。



     至此完成对blog本身的访问速度提升，当然，这是针对国内访问的，国外访问的话实际上用时会增加一些。
