访问速度优化(二)---国内解析到coding，国外解析到github

     之前通过将google字体和加载耗时较长的图片存到云对象存储(COS)大幅提升了这个网站的访问速度，其中google字体因为google的原因而耗时很长，而图片则是因为这个网站代码托管在github，需要从github那边加载，而github，国外的，访问速度相比对国内主机慢不少。
     因此将网站同时部署到github和coding，比较一下两个地方的ping值：

(1) ping github， 平均366.384ms

(2)ping coding， 平均34.388ms，ping值低10倍

     代码托管到coding和托管到github一样的流程，在coding注册账号后新建项目，不同于github pages的项目名必须和github用户名相同，在coding的项目名可以任意起，域名结构为：username.coding.io/projectname， 但若想通过username.coding.io访问，即不带项目名访问，则需要将项目名命名为用户名，如下图：

     然后部署，可以同时部署到coding和github，配置中添加一条部署仓库即可，以SSH连接为例：

     然而我在mac下deploy的速度很慢，不知原因，所以手动push，先在coding的账户->SSH公钥里添加公钥，直接添加github的公钥即可。

  • 在本地cd到放仓库的地方，然后clone下来，我的项目是git clone git@git.coding.net:louiehuang/louiehuang.git，clone下来的仓库就是louiehuang.git
  • 将generate生成的public文件夹下所有文件复制到仓库louiehuang.git
  • git add *，添加所有新加项，因为.gitignore的设置不用git add .，把这个ignore文件删掉后也可以用
  • git commit -m "comment"
  • git push origin master

     然后在coding中的项目中将Pages服务打开，将自己的域名添加进来，之后设置域名解析，将国内访问解析到coding，国外解析到github，在自己的云解析平台设置，如下图：

     至此完成了在coding的托管。因为每次更新都需要同时更新coding和github两个地方，所以写了段命令来bash，每次同时push到coding和github的仓库。
