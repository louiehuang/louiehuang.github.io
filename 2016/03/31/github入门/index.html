1. 先看看怎么在本地使用git

(2) 将这个文件添加到git仓库，分为两步

step1. git add test.md
step2. git commit -m "wrote a test file"

-m 后面接的相当于本次提交的注释，用于对提交内容进行说明

用法和mysql类似，同样是可以add多个后一起commit

(3) 尝试对test.md进行修改，这里在末尾增加一行内容：New content
可以通过git status 命令查看当前git仓库的情况

上述提示即test.md已被修改过，但还未被提交修改

(4) 查看文件的修改内容
git diff命令可以比较文件的不同内容，输入git diff test.md，输出以下提示：

即提示添加了”New content”

之后命令git add test.md, 输入git status查看当前情况，最后git commit -m "append new content"提交修改

(5) 版本回退
上一步之后，已经上git仓库里提交了两次test.md，第二次的test.md比第一次的多一行”New content”，通过git log查看此时版本情况

如果想退回到上一个版本的test.md，可以用git reset --hard HEAD^，退回上上个版本就是HEAD^^，虽然这里并没有… 退回前10个版本则可以HEAD10，此时再查看test.md，就是第一次编写的版本了。

那么如果没事找事又想恢复成第二版版本呢，可以通过上面那一长串的commit id来实现，这样 git reset --hard 892f0，只用输入一部分的id字符即可

如果关闭了终端，即忘记了上面的commit id的话，可以用git reflog查看，
