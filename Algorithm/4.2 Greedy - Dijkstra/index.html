4.2 Greedy - Shortest Path (Dijkstra)

Shortest Path - Dijkstra’s (Stays Ahead)

1. Problem Statment

Given a graph $G = (V,E)$ with $w(u,v) \ge 0$ for each edge $(u,v) \in E$ , find the shortest path from $s \in V$ to $t \in V$.

So Dijkstra’s Algorithm is used to find the shortest paths from a single source in a graph without negative weight edges.

  • If the graph has negative weights, we could use Bellman-Ford Algorithm.
  • if we want to find any pairs’ shortest path (multiple sources), we could use Floyd-Warshall’s Algorithm.

Both of these two algorithm will be covered in the later DP notes.


2. Solution

Greedy Approach. Choose the un-discovered node with shortest distance at each step.

  1. Start with a set $S$ of vertices whose final shortest path we already know.
  2. At each step, find a vertex $u \in V - S$ with the shortest distance from s.
  3. Add $u$ to $S$, and repeat.


3. Proof of Correctness

We’ll prove that at each step, Dijkstra’s algorithm finds the shortest path to a new node in the Graph.

Proof by Mathematical Induction (and contradiction):

1) General Statement P(n):
At each step $k$ (now $|S| = k$), Dijkstra’s algorithm finds the shortest path to a new node in the Graph.

2) Base Case Statement:
We prove $P(1)$ which says when $|S|=1$, Dijkstra’s finds the shortest path to a new node. This is apparently true since when $|S|=1$, $S = {s}$ and $d(s) = 0$.

3) Induction Hypothesis:
Assume $P(k)$ is true, i.e. when $|S| = k$ for each $k \ge 1$, Dijkstra’s finds the shortest path to a new node.

4) Induction Step:
Prove that $P(k+1)$ is true. We grow $|S|$ to size $k+1$, and prove we can find the shortest path to the new node. We can prove this by contradiction.

Consider the following example, by induction hypothesis, $P_u$ is the shortest $s \rightarrow u$ path for each $u \in S$. Now let anaylze $s \rightarrow v$ path $P$, we want to show that $s \rightarrow v$ path $P$ is the shortest from $s$ to a new node in $V-S$.

Suppose there is another path $s \rightarrow y$, both $y$ and $v$ are in $V-S$. There must be a node just before them in $S$ in order to construct such paths, name them $x$ and $u$ respectively.

Well, in $(k+1)^{th}$ iteration, we have $x$ and $u$ and are going to choose the next new node, the fact that we chose $v$ instead $y$ shows that the length of path $s \rightarrow v$ is shorter than or equal to that of path $s \rightarrow y$, and since all weights $\ge$ 0, we have $w(y,v) \ge 0$, thus path $s \rightarrow v$ is the shortest path discovered at this step.

Use PQ.

4.3 Time Complexity - $O(mlgn)$

  • Initiate PQ, using heapify, $O(n)$
  • Number of extractMin(), $n$ (n nodes)
  • Number of DecreaseKey(), m (m edges, so we will do this operation at most m times)

Thus total time complexity is $O(n) + O(nlgn) + O(mlgn) = O(mlgn)$.

Note:

  • If the graph is sparse, we have $m = O(n)$, thus $O(nlgn)$.
  • If the graph is dense, we have $m = O(n^2)$, thus $O(n^2lgn)$.

Also notice that if the graph is dense, we may want to use Fibonacci Heap. Remember that Fibonacci Heap only takes $O(1)$ for DecreaseKey(), insert() and merge(). Thus total time complexity is $O(n) + O(nlgn) + O(m) = O(m + nlgn)$. Since we have $m = O(n^2)$, total time complexity is $O(n^2)$.

4.4 Code

I use an additional array dist[] to store results just for convience - no heapify and no DecreaseKey(). But initiating the array is also $O(n)$; and in the updating step, though we do not need DecreaseKey(), we still need to add the newly discovered node into the PQ, this add() operation also takes $O(lgn)$ and will be called $m$ times, so $O(mlgn)$, thus total time complexity dose not change.
