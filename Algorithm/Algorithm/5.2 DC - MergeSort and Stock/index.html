2.2 Analysis

Although this problem can be even solved using Brute Force in $O(n)$ time, we are goint to look at how to use divide and conquer to solve it first.

If we divide the stock price array into 2 halves, called P1 and P2, then there are 3 possible cases that our max profit is achieved:

  • Both buy and sell the stock in P1 (left half)
  • Both buy and sell the stock in P2 (right half)
  • Buy in P1 and sell in P2.

Case 1 and case 2 can be solve recursively just like the original problem. For case 3, we need to find the minimum price min in P1 and maximum price max in P2, and (max-min) is the profit in this case. Compare the profits in these 3 cases, return the largest one.

The first two cases are computed in time $T(\frac{n}{2})$, each by recursion, and the third case is computed by finding the minimum in $P1$ and the maximum in $P2$, which takes time $O(n)$. Thus the running time $T(n)$ satisfies:

So the time complexity is $O(nlgn)$.

It seems that using divide and conquer can also achieve $O(n)$ time, but it looks like harder to implement.
