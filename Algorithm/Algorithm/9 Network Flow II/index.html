9 Network Flow II

Learning note for Algorithm Design (J. Kleinberg and E. Tardos).

Network Flow II

1. Bipartite matching problem

1.1 Problem Statement:

Find a matching M of largest possbile size in Bipiartite Graph G.

1.2 General Plan:

Design a flow network G’ that will have a flow value v(f) = k iff there is a matching of size k in G.

Moreover, flow f in G’ should identify the matching M in G.

1.3 Solution:

Run Max-Flow on $G’$. Say max flow is $f$. Edges carrying flow between set $X$ & $Y$ will correspond to our max size matching in $G$.

To prove this, we will show that $G’$ will have a max flow of value $k$ iff $G$ has a max size matching of size $k$.

A. Prove: If we have an $s \rightarrow t$ flow of value $k$ in G’, we can find a matching of size k in $G’$.

B. Prove: If we have a matching of size $k$ in $G$, we can find a $s \rightarrow t$ flow of value k in $G’$.

If we use ford-fulkerson algorithm, then $C$ here is $n$, so $O(nm)$, strongly polynomial.


2. Edge Disjoint Path

We first find a solution to finding the number of edge disjoint paths in a directed graph.

How do we modify this solution so it applies to undirected graph?

We can split the undirected edge to two directed edges, to construct G’, then apply the former solution to G’. There is one thing to take care of: If we use both of the two edges, then it actually means they are not helpful (we put 1 unit out and have 1 unit in, so they cancelled out, 0), so we can delete these two edges.

3. Circulation Flow (With just Demands)

Condition:

  1. Capacity Condition: for each egde $e$, $0 \le e \le c_e$
  2. Demand Condition: for every node $v$, $f^{in}(v) - f^{out}(v) = d_v$

Fact: If there is a feasible circulation with demands ${dv}$, then $\sum\limits{v}{d_v} = 0$.

We can: construct G’, create s and t.

If we find a flow whose value $v(f) = D$ in G’, then there is a feasible circulation in G.

Supply value $d_v$ in G is the same as having flow coming into it in G’ (source, $d_v$ < 0).


4. Circulation with Demands & Lower Bounds

Condition:

  1. Capacity Condition: for each egde $e$, $l_e \le e \le c_e$
  2. Demand Condition: for every node $v$, $f^{in}(v) - f^{out}(v) = d_v$

Solution:

Find feasible circulation (if it exists) in two passes. (The first pass is for lower bound and the second pass is for demand values)

Pass #1. Find $f0$ to satisfy all $l{e}$ ‘s

Pass #2. Use remaining capacity of the network to find a feasible circulation $f_1$ (if it exists)

Finally, combine the two flows: $f = f_0 + f_1$.

Flow imbalance value at node v: $Lv = \sum\limits{e\;into\;v}{le} - \sum\limits{e\;out \; of\;v}{l_e}$

Solution Details:

  1. Push flow $f_0$ through G where $f_0(e) = l_e$
  2. Construct G’ where $C^{‘}e = C_e - l_e$ && $d’{v} = d_v - L_v$ (after $f_0$ we already have $L_v$ for each node v, so there wil be $d_v - L_v$ still needed.)
  3. Find feasible circulation in G’, call it $f_1$
  4. If there is no feasible circulation in G’ => no feasible circulation in G; otherwise, feasible circulation in G = $f_0 + f_1$.


5. Min-Flow

Input:

  1. Directed Graph G(V,E)
  2. Source s
  3. Sink t
  4. $l_e$ for each edge $e \in E$
  5. There are no $c_e$’s

Solution:

  1. Assign “large” capacity (sum of all the lower bound) to all edges and find a feasible flow f.
  2. Construct G’, where all the edges are reversed and the reversed edge e has capacity = $f_e - l_e$ (we cannot undo more than $f_e - l_e$ because we need $l_e$)
  3. Find maximum flow f’ from t to s in G’
  4. Min-flow = f - f’.

Step 1, we have a feasible flow, but it is not necessarily the minimum flow, we need step 2 to send some flow “back”.


Reference:

[1] J. Kleinberg and E. Tardos. Algorithm Design. Pearson Studium, Boston, 2006.
