7.1 DP - Bellman-Ford

Dynamic Programming - Bellman-Ford

1. Motivation

We are takling about Single Source Shortest Path Problem.

Given a graph G with weighted edges, find the shortest path from s to t.

If all the weights are non-negative, we could use Dijkstra’s algorithm, time complexity is $O(|E|lg|V|)$.

Dijkstra’s algorithm does not work on graphs with negative weights. Consider following example, source is S and destination is A. Dijkstra will visit A first, thus S->A, but the real shortest path is S->B->C->A.

So we need other algorithm to deal with graphs that have negative weights.


2. Bellman-Ford

For Dynamic Programming, we need to somehow order the subproblems, otherwise we may get an exponential runtime.

Consider the path $u=w0,w_1,…,w{k−1},wk =v$. To have an optimal substructure, the sub-path $u=w_0,w_1,…,w{k−1}$ must be a shortest path to $w_{k−1}$.

Thus, we will be counting the number of edges in the shortest path. This is how we order subproblems.


Let $D[v, k]$ denote the length of the shortest pathfrom s to v that uses at most k edges.

Then the shortest from s to v either uses at most k edges (we will try all the neighbors of node v) or use at most (k-1) edges (if this is smaller).

  • Case 1: path uses at most k-1 edges.$D[v, k] = D [v, k-1]$.
  • Case 2: path uses at most k edges. check all vertices w adjacent to v, take edge (w, v), and then select the best s-w path that uses at most k-1 edges. $D[v,k] = \min\limits{(w,v) \in E}(D[w, k-1] + c{wv})$.

So our goal is $D[t, |V|-1]$ since path going from s to v needs at most |V|-1 egdes.


3. Implementation and time complexity

We can also implement this algorithm using 1-D array, just overwrite the result for k.

Time Complexity:

Notice that the inner two loops for v, e are just checking all edges, thus the time complexity for these two loops is $O(E)$. So the total runtime complexity is $O(VE)$


4. More Thoughts

Q: How to apply the Bellman-Ford algorithm to find out if a graph has a negative cycle?

A: We do not stop after (V-1) iterations, perform one more round. If there is such a cycle, then something should change in the table, thus we know there is a negative cycle.
