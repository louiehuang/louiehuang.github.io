4.3 Greedy - MST (Kruskal, Prim and Reverse Delete)

Minimum Spanning Tree (Exchange Argument)

1 Problem Statement

Find minimum cost network that connects all nodes in graph $G$.

Formal Statement:

“For certain pairs $(v_i, v_j)$, we may build a direct link between $v_i$ and $v_j$ for a certain cost $c(v_i, v_j) > 0$. Thus we can represent the set of possible links that may be built using a graph $G = (V , E)$, with a positive cost $c_e$ associated with each edge $e = (v_i, v_j)$.

The problem is to find a subset of the edges $T ⊆ E$ so that the graph $(V, T)$ is connected, and the total cost $\sum\limits_{e∈T}{c_e}$ is as small as possible. (We will assume that the full graph $G$ is connected; otherwise, no solution is possible.)”

Definition: Any tree that covers all nodes of a graph is called a spanning tree.

Definition: A spanning tree with minimum total egde cost is a MST.

So our Problem Statement becomes: Find a MST in an undirected graph.

2. Kruskal’s

Sort all edges in order of increasing cost. Insert each edge $e$ to $T$ in this order as long as it does not create a cycle with edges we’ve already added. If it does, simply discard this edge and continue.

So Kruskal’s algorithm will select edges ${(h,g), (c,i), (g,f), (c,f), (a,b), (c,d), (b,c), (d,e)}$.


3. Prim’s

Start with a node set $S$ (initially the root node) on which a MST has been constructed so far. At each step, try to greedily grow $S$ outward by one node - adding the node $v$ that minimizing the attachment cost (not the path cost from s to v).

This process is similar to Dijkstra’s Algorithm. (Prim is even simpler)

So Prim’s algorithm will select edges ${(a,b), (a,h), (h,g), (g,f), (f,c), (c,i), (c,d), (d,e)}$.


4. Reverse Delete

This is a backward version of Kruskal’s.

Start with a full graph, $(V,E)$ and begin deleting edges in order of decreasing cost as long as deleting it does not disconnect the graph.

So Reverse Delete algorithm will delete edges ${(d,f), (b,h), (e,f), (b,c), (i,h), (i,g)}$.


5. Cut Property

Cut Property: Assume that all edge costs are distinct. Let $S$ be any subset of nodes that is neither empty nor equal to all of $V$, and let edge $e = (v, w)$ be the minimum cost edge with one end in $S$ and the other in $V − S$. Then every minimum spanning tree contains the edge $e$.

Say, edge $(v,w)$ is the lowest cost edge that crosses the boundary between $S$ and $V-S$, we want to show that this edge belongs to every MST. Prove this by contradiction.

Assume that there is a MST $T$ that does not use edge $(v,w)$. Since MST is a network that connects all the nodes in the graph, there must be a way from v to w. Follow the edges on $T$ that go from w to v. Since this path goes from v to w, and $v \in S$ and $w \in V-S$, there has to be an edge that crosses the boundary between $S$ and $V-S$, name it $(w’, v’)$.

Remember we assume $(v,w)$ is not a part of MST, so adding this edge will create a cycle. The nodes on this cycle could be connected to one another either through $(v’,w’)$ or $(v,w)$. But $(v,w)$ has a lower cost, so if we replace $(v’,w’)$ with $(v,w)$, we reduce the cost of $T$, and doing so we still have a spanning tree since the nodes that are previously connected to each other through $(v’, w’)$ is still connected through $(v,w)$. Thus $T$ is not a MST, a contradiction.

5.1 Proof for Kruskal’s

Note: In the intermediate step, what Kruskal creates is a forest.

At each step, Kruskal’s algorithm picks an edge $(v,w)$ with minimum cost connecting $S$ to $V-S$. ($w$ does not belong to $S$, otherwise, adding this edge (from $S$ to $w$) will create a cycle. So $w \in V-S$, since $v \in S$, this edge connects $S$ to $V-S$.

And this edge must belong to a MST (by Cut Property). So in summary, at every step, Kruskal’s algorithm picks an egde that must belong to the MST, so at the end, the tree must be a MST.

5.2 Proof for Prim’s

Similar to proof for Kruskal’s, using Cut Property to show that edges Prim’s algorithm chooses at each step belong to a MST.

5.3 Proof for Reverse Delete

Cut property will not help us prove reverse delete since reverse delete focuses on the highest cost edges (Kruskal’s and Prim’s focus on lowest cost edges).

We need to prove “Highest cost edge in a cycle cannot belong to a MST”. (by contradiction)

For reverse delete, we are deleting the highest cost edge that does not disconnect the graph, meaning this edge must be the highest cost edge in a cycle. So Reverse Delete removes all the egdes that cannot be in the MST, the remaining part will be a MST.


6. Implementation

6.1 Kruskal’s

We need to find whether two nodes are in the same set quickly, and if they are not, we also want to merge them together quickly, these can be done using Union Find.

Union Find

Operation Array Implementation Pointer Implementation
Make set $O(1)$ $O(1)$
Find set $O(1)$ $O(lgn)$
Union $O(lgn)$ $O(1)$

So overall time complexity is $O(mlgm)$.

6.2 Prim’s

Just like Dijkstra’s algorithm.

Now we are not checking the distance from the source node $s$, we are only interested in finding the node in $V-S$ that has the shortest distance to the set $S$ (not source node $s$, but some node in the set $S$).

So overall time complexity is $O(mlgn)$.

So overall time complexity is $O(m^2)$.
