3.2 Heap

Heap

1. Binary Heap

A heap is a complete binary tree which satisfies the heap ordering property.

  1. min-heap: value(child) >= value(parent)
  2. max-heap: value(child) <= value(parent)

So by definition, there may exist duplicates in a binary heap.

1.1 Structure

We can use an array to represent the heap, to make operation more clear, we start array’s index from 1 instead of 0 (no big difference), so the root is located at index 1, its two children are located at index 2 and 3.

So consider a node at index $i$

  1. Its left child’s index is $2i$.
  2. Its right child’s index is $2i+1$.
  3. Its parent’ index is $\frac{i}{2}$.


1.2 Insert

Insertion can be done with two steps:

  1. Append the new element to the end of the heap array.
  2. Percolate it up to maintain heap property. (swap with its parent if needed)

1.4 decreaseKey

In this operation, we want to change the value of one of the heap node. For a min heap, we usually do not have the need to change a node’s value to a bigger one. So if we decrease a node’s value, we will percolate up this node.


1.5 Build

There are two ways to build a heap:

  1. Insertion, this takes $O(nlgn)$.
  2. Heapify, this takes $O(n)$.

If we cannot have all the elements (like receive elements one by one in a stream), then we may need to insert them to the heap one after another, each insertion takes $O(lgn)$, so this will take $O(nlgn)$ in total.

But if we have all the elements, like we are given an array to make it a heap, then we can do heapify. We start at position $n/2$ toward position $1$, and percolate each element down if needed. Let’s analyze this method’s time compelxity.

Consider a heap with height $h$, there is 1 node in the $h=0$ layer (root), 2 nodes in the $h=1$ layer… $2^{h-1}$ nodes in the $h-1$ layer and $2^h$ nodes in the last layer. root node will swap $h$ times in the worst case (down to the bottom), and nodes in the second to last layer will swap at most once.

height number of nodes number of swaps
0 1 h
1 2 h-1
2 4 h-2
… … …
h-1 $2^{h-1}$ 1

Sum them up, we have $T(n) = \sum\limits_{k=1}^{h}{k \cdot 2^{h-k}}$. Notice we also have $2^h = O(n)$.

So, $T(n) = \sum\limits{k=1}^{h}{k \cdot 2^{h-k}} = 2^h \sum\limits{k=1}^{h}{\frac{k}{2^{k}}} \le 2^h \sum\limits_{k=1}^{\infty}{\frac{k}{2^{k}}} = 2^h 2 = O(n)$.


1.6 Meld

We can merge two heaps in linear time: merge two arrays and then run heapify. $O(n)$


1.7 Summary

Opearation Complexity

insert $Θ(log n)$
decreaseKey $Θ(log n)$

Efficient searching is not supported. To search, we have to traverse the whole heap.

2. Binomial Heap

See: Binomial heap

The cost of inserting n elements into a binomial heap, one after the other, is $Θ(n)$ (amortized cost), even if n is not known in advance.


3. Fibonacci Heap

Idea: relaxed binomial heaps. (Lazy)

Goal: decreaseKey in O(1)

For decreaseKey():

Take the node needed to decrease, and change its key, and disconnect it and its entire subtree from where it is, and attach it to the tree root list. We do not fix the heap after decreasing. (We will fix the heap when deleteMin is called.)

For insertion():

Just add a single node to the top level. Do not merge binomial trees! We may have several trees of the same rank. We will fix the heap when deleteMin is called.
