4.1 Greedy - Interval Scheduling and Lateness

Greedy - Interval Scheduling and Lateness

An algorithm is greedy if it builds up a solution in small steps, choosing a decision at each step myopically to optimize some underlying criterion.

Two basic methods for proving that a greedy algorithm produces an optimal solution to a problem.

  1. Stays Ahead.
  2. Exchange Argument


1. Interval Scheduling (Stays Ahead)

Input: a set of requests, each request contains start s(i) and finish f(i) information.

Objective: find the largest compatible subset of these requests.

Output: Such subset.

1.1 Solution

Earliest finish time first (smallest f(i)) is a correct condition to pick request to get optimal solution. (Before this, we may want to try smallest s(i), smallest f(i)-s(i), but we can find counter examples for these strategies.)

1.2 Proof of Correctness

Two steps:

  1. Show that A is a compatible set.
  2. Show that A is an optimal set.
(1) Prove A is a compatible set

Every time this algorithm picks a request, it goes through all the rest of requests and delete any other request that is not compatible with the one it picks, so there is no possible way of picking request that are not compatible with each other.

(2) Prove A is an optimal set

A: $i_1, i_2, …, i_k$
O: $j_1, j_2, …, j_m$

If $A$ is optimal, suppose there is an optimal set $O$, then the size of $A$ must be equal to the size of $O$, i.e. $|A| = |O|$. (If $|A| < |O|$, then apparently, A is not optimal; if $|A| > |O|$, this is not possible, otherwise $O$ is not optimal, but we assume that $O$ is.)

=> So to prove $A$ is optimal, we are essentially proving that $|A| = |O|$.

=> We can prove this by contraction, If $|A| \ne |O|$, then $O$ must contain at least one more request than $A$, suppose $O$ has one more request, name it $j_{k+1}$.

A: …, $_{i{k}}\$
O: ….., $_{j{k}}\_$ , $_{j{k+1}}\$

Now let’s try to find a contradiction. Since O has one more request $j{k+1}$, if we can prove that A also has $j{k+1}$, (thus their sizes are equal), then we find a contradiction.

So, in what cases that A will have $j_{k+1}$?

Notice that in $O$, $jk$ is compatible with $j{k+1}$. So if finish time $f(ik) \le f(j{k})$, this means $i{k}$ will also be compatible with $j{k+1}$, thus A will choose $j_{k+1}$, so will have equal size as $O$.

=> Now the question is: can we prove that $f(ik) <= f(j{k})$? Let’s formalize this statement that we’re going to prove: For all $n \le k$, we have $f(i_n) \le f(j_n)$

=> A general way could be used to prove this kind of statment is through Mathematical Induction.

1) General Statement P(n):
For all $n <= k$, we have $f(i_n) \le f(j_n)$ where k is the size of $A$.

2) Base Case Statement:
We prove $P(1)$ which says $f(i_1) \le f(j_1)$, this is apparently true since our algorithm chooses request with the earliest finish time at each step.

3) Induction Hypothesis:
Assume $P(N)$ is true, i.e. $f(i_N) \le f(j_N)$

4) Induction Step:
Prove that $P(N+1)$ is true, i.e. prove that $f(i{N+1}) \le f(j{N+1})$ is true:

A: …, $__i{N}\_$ , $__{i{N+1}}\__$
O: ……, $__j{N}\_$ , $_{j{N+1}}\__$

Since $j{N}$ is compatible with $j{N+1}$ and the fact that $f(iN) \le f(j_N)$, we can infer that $i{N}$ is compatible with $j_{N+1}$.

Since we also know that $i{N}$ is compatible with $i{N+1}$ and the algorithm only choose the request with the earliest finish time, we could infer that $f(i{N+1}) \le f(j{N+1})$, otherwise, we will choose $j_{N+1}$.

Thus We have proved $P(N+1)$ under the assumption of $P(N)$.

Since the biggest number that n could be is k, by the Principal of Mathematical Induction, we conclude that P(n) is true for all n <= k.

=> Now we’ve proved that for all $n \le k$, we have $f(i_n) \le f(j_n)$.

Go back to the contradiction part, we know $|A|$ must be equal to $|O|$, thus $A$ is optimal.


2. Scheduling to Minimize Lateness (Exchange Argument)

“In contrast to the interval scheduling problem, however, each request in this problem is more flexible. Instead of a start time and finish time, the request $i$ has a deadline $d_i$, and it requires a contiguous time interval of length $t_i$, but it is willing to be scheduled at any time before the deadline. Each accepted request must be assigned an interval of time of length $t_i$, and different requests must be assigned nonoverlapping intervals.”

Suppose that we plan to satisfy each request, but we are allowed to let certain requests run late. Following is what we have in summary:

  • Requests can be scheduled at any time (we do not have start and finish time, but just length).
  • Each request $i$ has a deadline $d_i$.
  • Notation: $L_i = f(i) - d_i$, $L_i$ is the lateness for request $i$, $f(i)$ is the finish time, but we do not know it (it is determined by our assignment). We will say that $L_i = 0$ if request i is not late.
  • Goal: minimize the max Lateness $L = max_i{L_i}$, => $min{L}$

Greedy: Earliest deadline first

Solution: Schedule jobs in order of their deadlines without any gaps between jobs.

Def: Inversion: job $i$ is scheduled before job $j$, but $d_i \ge d_j$.

Proof steps:

  1. There is an optimal schedule that has no inversion and no idle time (gaps).
  2. All schedules with no inversions and no idle time have the same max Lateness.
  3. Schedule produced by our algorithm has no inverstions and no idle time. Thus it is optimal.

We can prove this by proving following facts:

Fact 1: There is an optimal solution with no gaps. (idle times)

This is obviously true, if there are gaps in an optimal solution, we can simply remove those gaps without increase max Lateness.

Fact 2: Jobs with identical deadlines can be scheduled in any order without affecting the max Lateness.

Fact 3: All schedules with no inversions and no idle time have the same max Lateness.

Fact 4: There is an optimal schedule that has no inversion and no idle time.

So we starting with an optimal solution, and by removing its gaps and inversions (if there are any) one by one, showing each step will not affect this solution’s optimality. This will finally lead to a same kind of solution that our algorithm produces (no gaps and no inversions), prove fact 3 to show our solution is also optimal.

“Our plan here is to gradually modify $O$, preserving its optimality at each step, but eventually transforming it into a schedule that is identical to the schedule $A$ found by the greedy algorithm. We refer to this type of analysis as an exchange argument.”


Reference:

[1] J. Kleinberg and E. Tardos. Algorithm Design. Pearson Studium, Boston, 2006.
