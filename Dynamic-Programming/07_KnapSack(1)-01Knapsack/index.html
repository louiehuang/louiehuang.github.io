0-1 Knapsack Problem

1. Description

Given n unique items,each with a weight and a value, determine the subset of items such that the total weight <= a given capacity W, and the total value V is as large as possible. You cannot break an item, so you can either pick the complete item, or don’t pick it (0-1 property).

(The fractional knapsack problem can be solved by a greedy algorithm. But 0-1 knapsack cannot)

For example, we have W = 10, and items:

Item Weight Value
1 6 30
2 4 14
3 1 9
4 2 12

Using greedy, we may choose item 1 and 2, thus have total value = 30+14, but in fact, we could choose item 1,3,4 thus have total value = 30+9+12.

2. Solution

Use $x_{k}$ to denote item k, if we choose it, $x_k=1$, otherwise, $x_k=0$, then for 0-1 Knapsack Problem, we are essentially trying to:

Let dp[k, w] be the maximum value we can achieve by using a knapsack of capacity $w \in [0,W]$ and with $k \in [1,n]$ items.

Consider item $k$, we could either choose it or not.

  1. If we choose item $k$, $dp[k, w] = dp[k-1, w-w_k] + v_k$, we can regard $w-w_k$ as make room for item $k$, so the total weight for (k-1) items is $w-w_k$.
  1. If we do not choose item $k$, then $dp[k,w] = dp[k-1,w]$

We just pick the bigger value of these two possible choices, also notice that when current available capacity is not enough for current item, i.i $w_k < w$, we cannot put it in, so $dp[k,w] = dp[k-1,w]$.

In the implementation above, be careful about the for-loop for updating w, we need to start from 0, try all valid capacity, check $wt[k-1] \gt w$ explictly:

This means we cannot code like this, start w from wt[k-1].

In the next optimization section, we use one-dimensional array, and we can use $wt[k-1]$, (for(int w = W; i >= wt[k-1]; w--), but here we cannot do that. Because what if(wt[k-1] > w) statement means is that if current capacity is not enough, we do not choose kth item, use previous result (k-1 items), in the two-dimensional array dp[][], this is the cell 1 position above it, we should copy that value down, so dp[k][w] = dp[k-1][w], but if we just start w from wk[k-1], we miss the case that wk[k-1] < w, thus dp[k][w] will remain as 0 instead of being updated to dp[k-1][w].

For example, val = {30,14,9,12}, wt = {6,4,1,2}, W = 10, when k = 4, w = 1 (consider whether item 3 should be put into the knapsack with capacity 1), look at dp[4][1] below, what should this ? be? It should be 9 (dp[3][1]), but if we start w from wt[k-1] = wt[3] = 2 > 1 = w, it will remain as the default 0 since we never run into the for-loop to update it.

(If we are using one-dimensional array dp[], when k = 4, now the ? is 9 not 0, since we overwrite this dp[] each time, from right to left)

Time complexity: $O(nW)$

Space complexity: $O(nW)$

2. Optimization - using a one-dimensional array

We can reduce space complexity from $O(nW)$ to $O(W)$. In the implementation above, we record all status for each item k, but this is not necessary, we can overwrite previous result, i.e. in the kth round, we overwrite the result of the (k-1)th round.

So this means we need to make sure that in the kth round, the value dp[w] before updating is the result of the k-1th round. To achieve this goal, we can decrement w from W instead of incrementing it. If we still increment w, then dp[w-wt[k]] is the value updated in the kth round (corrspond to dp[k,w-wt[k]]) rather than in the k-1 round (corrspond to dp[k-1,w-wt[k]]) .

Since we overwrite the array, w can stop at wt[k] instead of 0, we have for(int w = W; w >= wt[k]; w--).

3. Constructing the Optimal Solution

To compute the actual subset $T$, we can use an auxiliary boolean array $keep[k, w]$ which is 1 if we
decide to take the $k$-th item and 0 otherwise. So we can backtrack this array to output the answer.

  1. If $keep[k,w]$ is 1, then item k belongs to $T$, we then check $keep[k-1, W-w_k]$
  2. If $keep[k,w]$ is 0, then item k does not belong to $T$, we then check $keep[k-1, W]$

4. More thought - some details about initialization

  1. Sometimes we are required to fill up the knapsack ($\sum_{k=1}^{n}w_kx_k = W$, The total weight of items we choose should equal to W).
  2. Sometimes we do not have this requirement, we are just told to maximize the total value. (Like this classical 0-1 knapsack problem)

In the first case, we initialize $dp[0]$ to $0$, and $dp[1,…W]$ to $-\infty$.

Initializing by this way we can guarantee that when the solution we find is one of the optimal solutions that satisfys $\sum_{k=1}^{n}w_kx_k = W$.

For example, we will find {0,1} for the following input, thus have total value = 30+14=44. (We have {0,1} and {0,4} that can form total weight = 10, but first one has value 44, the second has value 35)

Why this initialization works?

We can regard dp array as the valid status when knapsack has no items in it. So if the knapsack needs to be filled up, then only the knapsack with capacity w=0 can be “filled up” when there is no item in it. All other capacityies do not have valid solution. These are “undefined” statuses, so we assign them with $-\infty$.

If the knapsack is not required to be filled up, then a knapsack with any capacity has a valid solution - “We put nothing into the knapsack”. This solution’s value is 0, so we initialize $dp[0…W]$ to $0$.
