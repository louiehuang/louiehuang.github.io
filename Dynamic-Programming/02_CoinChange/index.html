Coin Change

1. Minimum number of coins

Compute the minimum number of coins needed to make change for a given amount $m$. Assume that we have an infinite supply of each coin. All denominations $d_k$ are sorted in ascending order: $1 = coin_1 < coin_2 < … < coin_n$. ($coin_1= 1$ here is simply to make sure that there will always exist a solution, will discuss the case that no solution exists later on)

For this problem, Greedy Algorithm cannot always find the optimal solution. For example, we have m = 100, D = {1,5,50,80}, greedy that selects largest coin will produce {80,5,5,5,5}, 5 coins, while we just need 2, {50,50}.

1.1 Solution1 using $dp[][]$

Consider using DP. To count minimum number of coins, we can divide all set solutions in two sets.

  1. Solutions that do not contain kth coin $coin[k]$.
  2. Solutions that contain at least one $coin[k]$. Since we can use infinite same coin, we need to know the minimum number of coins to make up amount $x - coin_k$ by using first kcoins (including it), which is $dp[k, x-coin_k]$

Let $dp[k,x]$ be the least number of coins needed to represent amount $x$ ($0 \le x \le m$) using the first $k$ ($1 \le k \lt n$, k=0 is coin[0], the first kind of coin) kinds of coins. Then it can be written as the smaller one of the above two situations:

Therefore, the problem has optimal substructure property as the problem can be solved using solutions to subproblems.

  1. If m = 0, then the number should be 0, $dp[k, 0] = 0$
  2. If just use $coin[0] = 1$, then the number is our target amount, $dp[0,x] = x$.
  3. If the amount needed now is less the the current coin k, i.e. $x < d_k$, $dp[k, x] = dp[k-1, x]$
  4. $dp[k,x] = min{dp[k, x-coin_k] + 1, dp[k-1, x]}$

2. Solution2, optimize memory using $dp[]$

In the above solution, we used a two-dimensional array to store our intermediate results. We considered two variables, k and x, use the first k kinds of coinds to represent amount x, thus have dp[k, x]. But $k$ is not really necessary. => $dp[x]$ is enough.

There are two ways to think of it.

(1) The first way. Still follows the thought of using the first k kinds of coins to represent amount x (pick coins one by one). But instead record every results for each k, we overwrite the results of $(k-1)$. This is correct because in kth round, we have the optimal solution to using $(k-1)$ kinds of coins like in solution 1, we can just consider whether use kth kind of coin or not. Let dp[x] denote the minimun number of coins needed to represent x.

Here, $dp[x]$ in the $min{}$ is actually the result in $(k-1)$ round.

(2) The second way. We just consider how we can represent $dp[x]$ with its subproblems. For $dp[x]$, we either chose a particular coin - coins[k] or not. So what is the k? 3. If solution to this problem does not exist

Previous analysis assumes that $coin[0] = 1$, thus there must be a solution to any positive amount m. Now we consider the case that a solution does not exist. For example, m = 7, coins = {3,5}

To solve this problem, we just need to modify solution 2 slightly. Instead of initializing dp[value] = value, we initialize dp[value] = Integer.MAX_VALUE (since we are trying to find minimum). Then in the updating step, if dp[value - coins[j]] == Integer.MAX_VALUE , we do not update dp[value]. The first time that dp[value - coins[j]] != Integer.MAX_VALUE is that value = coins[j] (dp[0] = 0).

2. Total number of ways to make the change

We want to make change for $m$ cents, and we have infinite supply of $coins = {coins_1, coins_2, .. , coins_n}$ . How many ways can we make the change? The order of coins doesn’t matter.

For example, for m = 4 and coins = {1,2,3}, there are 5 solutions: {1,1,1,1},{1,1,2},{2,2},{1,3}. For m = 10 and coins = {2, 5, 3, 6}, there are 5 solutions: {2,2,2,2,2}, {2,2,3,3}, {2,2,6}, {2,3,5} and {5,5}.
